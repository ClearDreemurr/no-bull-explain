## 减治型递推式

> 用户提问："好。T(n) = T(n-1) + Θ(n) 这又是个什么玩意"

### 和主定理的区别

这个和前面画风完全不同。

主定理管的是"把活儿砍成几等份去递归"——但 `T(n) = T(n-1) + Θ(n)` 是**削掉一个、递归剩下的**，不是等分。

### 低配场景

你在数一段台阶的走法，但策略很朴素：

> 要做 n 阶的事，先干一坨 Θ(n) 大小的活，然后留给下一个兄弟去处理 n-1 阶的事。那个兄弟也如法炮制，再做一坨 Θ(n-1) 的活，交给下一个……

说白了就是一个**递归剥洋葱**：每层剥掉一片，剩下的继续剥。

### 朴素展开：递归链，不是递归树

```
T(n) = T(n-1) + n

第 0 层:  T(n)   = T(n-1)   + n
第 1 层:         = T(n-2)   + (n-1) + n
第 2 层:         = T(n-3)   + (n-2) + (n-1) + n
第 3 层:         = T(n-4)   + (n-3) + (n-2) + (n-1) + n
...
第 k 层:         = T(n-k)   + (n-k+1) + ... + (n-1) + n
```

它没有"分叉"，只有一条**单链**——不像归并排序每层指数级增殖。

当 k 一直走到 n 时，`T(n-k)` 变成 `T(0)`——常数基准情况。

```
T(n) = T(0) + 1 + 2 + 3 + ... + (n-2) + (n-1) + n
     = n(n+1)/2
     = Θ(n²)
```

### 映射回真实机制

| 符号 | 动作角色 |
|------|----------|
| `T(n)` | 解决规模 n 的总时间 |
| `T(n-1)` | 把规模砍掉 1 后，剩下的递归开销 |
| `Θ(n)` | 砍掉这 1 个单位时，要额外做的线性扫荡工作 |
| 递归链 | 从 n 一路降到 1，没有分支 |

这种"每次减 1、线性扫描"的递归模式，在算法里对应的东西：
- **选择排序**：每次找最小值（O(n)），递归排剩下的 n-1 个 → T(n) = T(n-1) + O(n)
- **插入排序最坏情况**：每次要把新元素插到最前面（O(k)），累计 = 1+2+...+n = Θ(n²)
- **快排最坏情况**：每次划分出 1 和 n-1 → T(n) = T(1) + T(n-1) + n → Θ(n²)

### 去魔法化定义

> `T(n) = T(n-1) + Θ(n)` = 每次只削掉 1 个元素，削每个元素要线性扫描一遍剩下的，总开销等于 1 加到 n，是 **Θ(n²)**。

这种"不做分治、只做递减"的递归，主定理管不着，必须自己展开成等差数列。每次减 1 的线性加和 ≈ n²/2。

> 用户回复1："哦哦我懂了相比主定理：对于每一层，都可以把它拆成b个子问题，每个子问题的规模比原来小了a倍，解决完所有子问题后把子问题合并到这一层需要花费f(n)，这个问题强调：每一层都可以把它拆成规模少1的一个子问题，解决完子问题后合并到这一层花费Θ（n）对吗"

你的整体方向是对的，但 a 和 b 的角色说反了。

> 用户回复2："就是说反了是吧，子层是T(n/b)，说明规模缩小了b，外头套一个a，说明拆成了几个？"

对，完全正确。

```
T(n) =  a  ·  T(n/b)  +  f(n)

```

| 符号 | 你误说成 | 实际 |
|------|---------|------|
| **a** | "小了a倍"的倍数 | **同时追几份**（比如 2T(n/2) 就是拆 2 份同时追） |

标准说法：拆成 **a** 个子问题，每个子问题的规模缩小到原来的 **1/b**。

## 递减型：T(n) = T(n/2) + Θ(1)

> 用户回复："好，那T(n) = T(n/2) + Θ（1）是不是说，每一层规模小2/1，每一层花费常数时间处理。这是二分查找吗？"

几乎全对，就一个小笔误帮你纠正：

```
T(n) = T(n/2) + Θ(1)
         ↑         ↑
     每份缩到1/2   每层花常数时间（比如比一次大小）
```

你写的是"小**2/1**"——那是 2 倍，不是 1/2。你是想说 **1/2**（缩一半）对吧？这个笔误不重要，你已经说对了意思。

**对，这就是二分查找。**

- 每层规模缩到 1/2 ✅
- 每层花常数时间处理 ✅（比一下就扔掉另一半）
- a=1（只追1个分支） ✅

回到主定理标准：
- a=1, b=2 → n^(log₂1) = n⁰ = 1
- f(n) = Θ(1)，和 n⁰ = 1 一样快
- Case 2 → **Θ(log n)**

## 关于"时间复杂度"和"解"的说法

> 用户回复："好，我看abf形式的题目问的都是时间复杂度，后面这两道Θ题目问的都是解，有什么区别？为什么问的不同？"

在算法分析的场景下，**它俩是一回事。**

"解这个递推式" = "求出这个递归算法的时间复杂度"。得到的都是同一个东西：**Θ(n log n)** 或 **Θ(n²)** 或别的渐进界。

**有主定理时：** 你直接套用 n^(log_b a) vs f(n) 的比较，得到结论。过程像查表，结果叫"时间复杂度"。

**没有主定理时（比如 T(n-1)）：** 你得手动展开求和。这个过程更接近"数学推导"，所以叫"求解"——因为你在做真正的数学展开。

但两个动作的最终产物，是同一个东西：

```
        T(n) = T(n-1) + n
              ↓ 手动展开
        T(n) = n + (n-1) + (n-2) + ... + 1
              ↓ 求和公式
        T(n) = n(n+1)/2
              ↓ 取最高阶
        Θ(n²)                    ← 这个既是"解"也是"时间复杂度"
```

### 后续追问

> "那后面这种过程怎么写？"

**标准三步展开法：**

**例：** T(n) = T(n-1) + n，T(1) = 1

**第 1 步：展开前几项，看出规律**
```
T(n) = T(n-1) + n
T(n-1) = T(n-2) + (n-1)
T(n-2) = T(n-3) + (n-2)
...
```

**第 2 步：往回代，直到基准**
```
T(n) = T(n-1) + n
     = [T(n-2) + (n-1)] + n
     = [T(n-3) + (n-2)] + (n-1) + n
     = ...
     = T(1) + 2 + 3 + ... + (n-1) + n
```

**第 3 步：求和，取最高阶**
```
T(n) = 1 + 2 + 3 + ... + (n-1) + n
     = n(n+1)/2
     = Θ(n²)
```

**另一个例子：T(n) = 2T(n-1) + 1（每次加倍）**
```
T(n) = 2T(n-1) + 1
     = 2[2T(n-2) + 1] + 1         = 4T(n-2) + 2 + 1
     = 4[2T(n-3) + 1] + 2 + 1     = 8T(n-3) + 4 + 2 + 1
     = ...
     = 2^(n-1)·T(1) + 2^(n-2) + ... + 2 + 1
     = Θ(2^n)
```

**模板总结：**
```
① 展开：         T(n)   = T(n-1) + n
② 迭代回代       = T(n-2) + (n-1) + n
  直到基准        = ...
                 = T(1) + 2 + 3 + ... + n
③ 求和转封闭形式  = n(n+1)/2
④ 取最高阶       = Θ(n²)
```

## 不均匀划分：1:9 分治

> 用户回复："我自己写好了，不过我还看到一个类似的问题是若每次划分都产生1：9的分割（即一个字问题大小为1/10，另一个为9/10)，则其时间复杂度满足的递推式解为?这又是啥"

每次把数组分成两份，一份是 1/10，一份是 9/10：

```
正中间
    ↓
[ 1/10 | ← 9/10 → ]
```

对应的递推式：

```
T(n) = T(n/10) + T(9n/10) + Θ(n)
```

主定理要求所有子问题规模相同（a·T(n/b) 形式）。这里两个子问题规模不同（n/10 和 9n/10），所以直接套不了。

### 解法：递归树法

还是把它展开成树——但这次两边不对称。

```
第 0 层:                [n]                       总花销 = n
                      /    \
第 1 层:          [n/10]  [9n/10]                 总花销 = n/10 + 9n/10 = n
                  /  \      /    \
第 2 层:     [n/100] [9n/100] [9n/100] [81n/100]  总花销 = n/100×2 + 9n/100×2 + 81n/100 = n
               ...     ...     ...     ...
```

每一层的总花销加起来都是 n。

### 树有多深？

两边深度不一样：

```
左分支（快）：每次缩到 1/10
  1 → 1/10 → 1/100 → 1/1000 → ...
  深度 = log₁₀(n)

右分支（慢）：每次缩到 9/10
  1 → 9/10 → 81/100 → 729/1000 → ...
  深度 = log₁₀/₉(n)
```

树的总深度按最慢的分支算：

```
总深度 = log_{10/9}(n)
```

log_{10/9}(n) 的量级：

```
10/9 ≈ 1.111...

log_{1.111}(n) = ln(n) / ln(1.111) ≈ ln(n) / 0.105 = Θ(log n)
```

和普通的 log₂n **是同一个量级**——都是 **Θ(log n)**，只是常数倍的区别。

### 结果

```
每层工作量 = n
总层数     = Θ(log n)
总工作量  = n × Θ(log n) = Θ(n log n)
```

**结论：T(n) = T(n/10) + T(9n/10) + Θ(n) 的解 = Θ(n log n)**

这是一个经典结论：**QuickSort 在绝大多数不均匀划分下，复杂度仍然是 O(n log n)，不是 O(n²)。**

### 和极端情况对比

| 划分比例 | 递推式 | 总复杂度 |
|----------|--------|----------|
| 1:1（完美对半） | 2T(n/2) + n | Θ(n log n) |
| 1:9 | T(n/10) + T(9n/10) + n | Θ(n log n) |
| 1:n-1（最坏） | T(1) + T(n-1) + n | Θ(n²) |

只要划分不是"削掉一个剩一堆"这种极端，就算 1:9，也还能保持 O(n log n)。只有退化为 1:n-1，深度 = n，才是 Θ(n²)。

## 为什么不能近似成单分支套主定理


> 用户回复："你是说可以完全认为T(n) = T(9/10 n) + n吗？这时候可不可以用主定理公式"

**不能。**

如果假装 `T(n) = T(n/10) + T(9n/10) + n ≈ T(9n/10) + n`，然后套主定理：

```
a = 1        → 只追 1 个子问题
b = 10/9     → 问题规模缩到原来的 9/10
f(n) = n

n^(log_b a) = n^(log_{10/9} 1) = n⁰ = 1

f(n) = n  远大于 1  →  Case 3
结论：Θ(n)
```

但正确答案是 Θ(n log n)。**近似结果是错的。** 为什么？

### 两个模型摊开比

**两分支版本（真实的）：**
```
                      [n]                     ← 这层花 n
                    /      \
第 1 层:        [n/10]    [9n/10]             ← 花 n/10 + 9n/10 = n
                /    \    /       \
第 2 层:  [n/100] [9n/100] [9n/100] [81n/100] ← 还是 n
              ...     ...     ...      ...

每层总花销 = n
深度 = log_{10/9}(n)  ← 最慢的那条路
总和 = n × log_{10/9}(n) = Θ(n log n)
```

**一分支近似版本（单链）：**
```
[n] → [9n/10] → [81n/100] → [729n/1000] → ...

第 0 层:  n
第 1 层:  9n/10
第 2 层:  81n/100
第 3 层:  729n/1000
...               ← 每层越来越小，是等比数列！

n + 0.9n + 0.81n + 0.729n + ... = n/(1-0.9) = 10n = Θ(n)
```

### 区别的本质

```
两分支版本：每层总花销 = n，有 log n 层 → n × log n
一分支版本：每层花销递减，总和 = 10n   →       n
```

**丢失的 log 因子，全来自那些被扔掉的左分支。**

只看最右这条链是一条线；加上左分支后，节点数指数级增长，虽然每个节点越来越小，但它俩刚好抵消——**每层总大小恒为 n**，所以每层花销恒为 n。

而一分支版本里，你走的那条链子越走越小，没有新的节点补充进来，花销每层都在缩。

### 核心教训

在递归树中，如果你忽略一个分支，你同时丢了所有从它长出来的子节点。

- 两分支版：每层问题总规模 = n，因为所有碎片合起来还是 n → Θ(n log n)
- 一分支版：只跟一个孩子走，越走越小 → Θ(n)

### 关键分界线

| 划分比例 | 能不能近似成一条链 | 原因 |
|----------|-------------------|------|
| **1:9** | ❌ 不行 | 两个分支深度不同，但每层总规模恒为 n，多了 log 因子 |
| **1:n-1**（极端） | ✅ 可以 | 大的那个分支占了绝对主导，小的可以忽略——`T(n) ≈ T(n-1) + n`，结果确实是 Θ(n²) |

### 后续追问

> "如果是n-1是不是就变成T(n) = T(n-1)+n，上面那个等差数列问题了，排序问题？"

完全正确。

| 递推式形状 | 复杂度 | 对应场景 |
|-----------|--------|---------|
| T(n) = T(n/10) + T(9n/10) + n | Θ(n log n) | 快排平均情况（1:9） |
| T(n) = T(1) + T(n-1) + n | Θ(n²) | 快排最坏情况（1:n-1） |

而 `T(n) = T(n-1) + n` 就是你刚才亲手展开的那个等差数列——**每次削一个，削一个扫一遍**，结果是 Θ(n²)。这正是快排最坏情况（数组已有序、每次选第一个当 pivot）。

## 和主定理对比表

| 特征 | `T(n) = aT(n/b) + f(n)` | `T(n) = T(n-1) + Θ(n)` |
|------|------------------------|------------------------|
| 怎么递归 | 等分成 b 份，同时追 a 份 | 削掉 1 个，只追 1 份 |

| 计算公式 | 主定理的三个 Case | 直接累加 1 到 n |
| 典型复杂度结果 | Θ(n log n) 或 Θ(n^log_b a) | Θ(n²) |
| 适用场景 | 归并、快速、二分 | 选择排序、插入排序最坏 |
